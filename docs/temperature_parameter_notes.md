# Temperature 参数笔记

## 1. Temperature 参数的作用
- **定义**：控制模型生成文本的随机性
- **取值范围**：0.0 - 2.0（通常使用 0.1-1.0）
- **影响**：
  - 较低值（0.1-0.3）：生成结果更确定、更保守
  - 中等值（0.7）：平衡随机性和连贯性
  - 较高值（1.0+）：生成结果更具创意，但可能更不稳定

## 2. 对模型耗时的影响
- **一般规律**：temperature 与耗时成正相关
- **原因**：
  - 较高的 temperature 会让模型在每个生成步骤考虑更多可能的 token
  - 模型需要计算更多概率分布，增加了计算复杂度
  - 生成较长文本时，这种差异会更明显

## 3. 对 Tokens 消耗的影响
- **通常情况**：temperature 与 tokens 消耗也成正相关
- **原因**：
  - 较高的 temperature 可能导致模型生成更长的文本
  - 更随机的生成过程可能产生更多"发散"的内容
- **注意**：这种关系不是绝对的，还取决于具体的模型和任务

## 4. 最佳实践
- **准确性任务**（如问答、总结）：使用较低的 temperature（0.3-0.5）
- **创意任务**（如写作、头脑风暴）：使用较高的 temperature（0.7-1.0）
- **平衡需求**：使用中等 temperature（0.5-0.7）

## 5. 测试建议
- 修改 `scripts/compare_models.py` 脚本，为每个模型测试多个 temperature 值
- 记录不同 temperature 下的响应时间和 tokens 消耗
- 比较结果以获得更准确的结论

## 6. 总结
在其他因素固定的情况下，temperature 通常与模型耗时和 tokens 消耗成正比，但具体影响程度会因模型、任务类型和生成长度而异。
